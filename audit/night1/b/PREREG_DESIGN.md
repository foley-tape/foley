## 0. 问题重述

输入：一条 coding agent 的会话日志流（JSONL、append-only、实时尾随）。
输出：(1) 一根连续的**张力指针** T(t) ∈ [0,1]；(2) **极少数**离散声音事件（≤5 种词汇量）。
目标：让人"听见"一场编码会话的情绪弧线——而不是听见日志本身。声音的稀有性是产品价值的一部分：默认状态是安静，指针在动。

## 1. 总体架构（D1–D4）

- **D1 分层管线**：尾随器(tailer) → 适配器(adapter：解析+归一) → 小型事件代数 → 特征提取（纯函数、窗口化）→ 张力引擎（确定性状态机 + 积分器）→ 发声策略（阈值/迟滞/预算）→ 渲染器（音频）。每层单向依赖，渲染器可整体替换。
- **D2 引擎是事件流的纯函数**：同一磁带重放 → 比特级相同的输出事件序列。live 与 replay 共用同一引擎，唯一差别是时钟源。这是全部金测试的地基。
- **D3 适配器永不崩**：坏行跳过并计数；半行（无换行符结尾）等待续传；非 UTF-8 lossy 解码；单行长度上限（如 1MB），超限只取截断头部里的结构字段。适配器的鲁棒性预算应当高于引擎——它面对的是外界。
- **D4 时钟纪律**：引擎内部只认**事件时间戳 + 固定步长 tick**。live 模式由墙钟驱动 tick，但 tick 时刻量化到固定栅格（如 100ms），调度抖动不得进入状态。

## 2. 事件怎么筛（D5–D9）

- **D5 事件代数尽量小**：`AgentText / ToolCall{kind} / ToolResult{ok|err, errClass} / UserMsg / Silence(由tick导出) / SessionStart|End`。工具归入桶：`read / search / edit / exec / net / other`——引擎只见桶，不见工具名。
- **D6 白名单原则**：不认识的记录类型一律丢弃并计数，不猜测语义。
- **D7 去抖与合并**：同类事件 250ms 内合并；密度信息由特征层窗口聚合给出，引擎不直接消费原始事件密度。
- **D8 "预期内失败"降权**：失败权重按工具桶区分——exec/test 失败权重 1.0，search 未命中 0.1，read 不存在 0.2。grep 的 exit 1 不是危机。
- **D9 用户信号单列**：用户打断（agent 输出途中插话）与用户纠正（短消息 + 否定语气）是强信号，独立特征通道，不与工具错误混算。

## 3. 张力怎么算（D10–D16）

- **D10 双层结构**：特征 → 目标张力 target(t)；状态 T(t) 以非对称时间常数追 target：升快降慢（τ_rise ≈ 5–10s，τ_fall ≈ 30–60s）。焦虑来得快，散得慢，这符合体感。
- **D11 target 是少数特征的软饱和和**：`target = tanh(Σ wᵢ·fᵢ)`。特征只取：错误 EWMA、同类错误重复度（第 n 次重复按 ~n^1.5 超线性）、edit→exec→fail 循环检出、高张力下的长停顿(stall)、用户纠正。特征 ≤6 个，多了不可调。
- **D12 离散积分必须定步长**：dt 设上限（如 250ms），大间隔切片积分；指数衰减/弹簧**优先用解析解**（闭式），至少半隐式积分——显式欧拉 + 巨大 dt = 爆振，这是最经典的翻车点。事件断流后一次性灌入几小时的 dt 绝不允许一步吞下。
- **D13 数值域有界**：T 全程夹紧 [0,1]；一切特征累积量用有界窗口，**无终身累加器**（防浮点漂移与长跑溢出）。
- **D14 确定性**：固定求和顺序；无种子不用随机；时间只用整数毫秒；接受 IEEE754 double 但金测试跨平台断言零差异——若做不到，在输出层量化（如 1e-6）后再比。
- **D15 校准期**：会话前 ~20 事件或 ~2 分钟只动指针不发声（baseline 校准，防冷启动误报）。
- **D16 episode 边界**：长间隔（如 >5 分钟无事件）按 episode 边界处理：状态重置或折半，**不跨边界连续积分**。跨午夜/系统睡眠属于此类。

## 4. 怎么防误报（D17–D23）

- **D17 一切离散触发用迟滞双阈值 + 驻留时间**：如进 0.7 / 出 0.5，dwell ≥ 10s；**等号归属显式写死**（≥ 进、≤ 出），并有金测试样本恰好落在阈值上。
- **D18 佐证要求**：危机级声音需要 ≥2 个独立特征同时超线；单特征只推指针，不发声。
- **D19 声音预算**：每类声音 cooldown（≥60s）+ 全局令牌桶（如 10 分钟 ≤3 声）。宁静默，不误鸣——沉默是安全默认值。
- **D20 errClass 归一**：去路径/行号/hex/数字/时间戳后取错误首行签名，用于**重复检测**；但对外类别是粗枚举（≤12 类），归一失败落 `other`。归一是双刃剑：过度归一并掉不同错误，欠归一拆散同一错误——两个方向都要有测试。
- **D21 repWindow 用事件时间不用墙钟**：窗口边界的含头含尾语义写进金测试（差 1ms 的样本对）。
- **D22 静默≠危机、断流≠dt**：无事件只导致缓慢回落；dt > 5 分钟按 episode 边界处理而不是当积分间隔灌入。
- **D23 可审计性**：每次发声携带"为什么"（触发时特征快照），回放时能审计每一声的成因——误报分析不靠猜。

## 5. 怎么表达"解脱"（D24–D28）

- **D24 解脱是边沿不是状态**：仅当"持续高张力（T>θ_high 且 ≥ dwell）→ 出现**正向事件**（测试通过/任务完成）→ 短窗内跌破 θ_low"三段齐备，才发 RESOLVE。
- **D25 挣来的解脱 ≠ 不了了之**：无正向事件、纯靠时间衰减降下来的，是另一种更弱更含糊的声音（呼气式/淡出式）。两者必须**听得出区别**——这是情绪语义的核心区分。
- **D26 指针本身参与表达**：允许轻微欠阻尼过冲——下探再回稳，物理上像"叹气"；若有磁带隐喻，wow/flutter 随张力升降、解脱时缓慢回到标称速。
- **D27 解脱后不应期**（如 2 分钟）：抑制新的危机声，给"余韵"，也防止震荡边沿连续触发。
- **D28 高张力中会话戛然而止**：专门处理（静默或专用弱音），绝不误发 RESOLVE——"没解决就结束了"是真实且常见的结局。

## 6. 隐私蒸馏（D29–D33）

- **D29 威胁模型：任何用户串都是放射性的**。蒸馏带只含：事件类型枚举、粗粒度相对时间、工具**桶**、errClass **枚举**、量化数值。**白名单 schema、零自由文本字段**——靠"净化黑名单"（识别并抹掉秘密）的方案默认不通过，因为黑名单永远数不完。
- **D30 工具名是用户数据**：MCP 自定义工具名可含公司名/项目名/敏感词，蒸馏时映射到固定 vocabulary，未知 → `other`。
- **D31 时间可指纹化**：绝对时间戳去除，只留量化相对 delta（如 1s 桶），会话起点加随机偏移；事件间隔序列本身仍构成残余指纹风险——至少要在文档里声明。
- **D32 短密钥问题**：任何"按长度阈值识别 token"的净化（如只抓 ≥16 位）天然抓不住 8–12 位口令/PIN/内嵌参数（`-pSECRET`）。若必须保留错误签名用于重复检测，只存加盐后弃盐的不可逆哈希。
- **D33 机械红线自测**：CI 金测试对蒸馏输出全字段做"不含任何源带子串（长度≥6）"的机械校验。人审会漏，机器不会累。

## 7. 测试与验收（D34–D37）

- **D34 金磁带**：每盘配期望时刻表（声音+毫秒时刻+特征快照），全量比对。
- **D35 性质测试**：同毫秒事件重排不变性（或定义稳定排序键）；重放确定性双跑 diff；恶意行 fuzz 不崩。
- **D36 长跑**：6–8h 合成流，RSS 有界、tick 漂移有界（无终身累加器是前提）。
- **D37 跨平台**：≥2 个 Node 版本双跑 diff 零差异，或显式声明量化容差。

## 8. 最可能翻车的五处（写给自己的预警）

1. 大 dt 一步积分 → 弹簧爆振（D12）。
2. 长度阈值式脱敏抓不住短密钥；黑名单式净化的固有漏面（D29/D32）。
3. 阈值等号与窗口边界的 off-by-one（D17/D21）。
4. 工具名/URL/命令参数直通蒸馏带（D29/D30）。
5. live 与 replay 时钟源不一致，长出两个行为世界（D2/D4）。
