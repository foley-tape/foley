## 0. 先给结论

Foley 的机器本体已经不是概念稿：真实晚到会话里，首手势后有单一 AudioContext、有声、有纸、有状态；杀源、杀服务与同标签恢复也能诚实呈现。旧 P0「接线倒置」与「暂停双声叠加」本轮均守住，不应重开。

但公开产品仍被四件事截断：

1. README 的 `npx foley` 在干净环境直接 E404，`private:true` 仍在，发布前 README 计数闸也会失败。
2. README 承诺“不存对话文本”，实现却读取、持久化并回显首条真人发言的最多 80 字符片段；“只在本地”不能替代“不存”。
3. 零会话新手首屏是一台装着死 LIVE、显示 `SIGNAL LOST`、同时递来接线单的机器；首分钟没有价值证据。
4. 直属真实 Claude 进程已经终止至少 26 秒，Foley 仍亮 REC、无告警；它把“尾随器活着”误当成“agent 活着”。

发布顺序应是：**先修信任与入口，再修首分钟，再谈更多声画、适配器或分享面。**

## 4. 产品判断：Foley 到底该是什么

一句话边界：

> **Claude FM 陪你，Agent View 盯它；Foley 把 agent 劳动变成“不看屏幕也能感到、第二天仍留在桌上”的本地物证。**

它不该争“更好的 agent 音乐播放器”，也不该争“更多 agent 的控制台”。Claude Code 用户已经能用 radio、hooks 与状态视图；Codex 用户也能跨设备审阅/批准任务（[官方说明](https://openai.com/index/work-with-codex-from-anywhere/)）。Foley 唯一值得守的楔子是：**环境存在感 + 需要你时的可信呼唤 + 介质留下的历史。**

### 4.1 唯一主线

只做这一幕：

`agent 工作 → 需要你 → 被你/自己解开 → 完成或停下 → 次晨留下一张可读的纸`

任何功能若不让这条链更可信、更快被懂或更值得次日回来，暂缓。

### 4.2 必砍 / 冻结

- **砍 hosted replays。** 它扩大信任、身份、存储与分享边界，却不增强 Foley 的本地楔子。
- **冻结 weather→选曲、texture 分音与更多唱片层玩法。** 当前不是声音花样不足，而是入口、信任与首分钟断裂。
- **更多 agent adapter 不先做。** Codex adapter 只在 hook-native 真相桥与新手路径稳定后再开；否则只是把坏首分钟复制到第二个平台。
- **不再扩 record library。** 三首够证明系统；更多曲目不会改变留存。
- 冰箱 #4 应按现状结案并重写：默认脱敏与导出机制已存在；只保留“经确认的本地分享物”意图，不复活托管分享。

## 7. 宪法挑战：向“绝对禁蓝”开火

### Steelman

《琥珀宪法》光法禁止一切冷白与蓝光（`琥珀宪法.md:17`）。这条禁令能挡住当代 SaaS 的霓虹青蓝、廉价 RGB 与“科技感”捷径，让暖灯、黄铜、纸和牛血红维持同一个年代；对一个极易滑向赛博仪表盘的项目，它是有效的审美防火墙。

### Fire

绝对禁令把“无物理理由的数字蓝”与“有物理来源的蓝色放电”混成一类。当前 CUE 已被实现为氩气放电管的临床蓝（`plate.css:218`，账本亦有修正记录），这是器件物理与“需要你”的单义信号，不是网页装饰。正文不修、靠后置账本开例外，会让宪法与现役机器永久互相指控，也削弱“历史诚实”本来要保护的东西。

### 建议修宪文本
