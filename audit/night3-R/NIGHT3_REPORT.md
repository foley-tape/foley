### 1.3 失效三式·死相册

1. **杀 claude**（停喂替身）：床转近隐（RMS 0.007≈马达低哼独存）、盘停灯暗、翻牌留白——机器**诚实转入待机，不装死不谎叫**。但 `body.rec-live` 不撤：**REC 红呼吸继续**。「安静的会话」与「死掉的 agent」在机器语言里**同相不可分辨**（对 agent 死亡，机器无自有判据，这可以辩护；但 REC 的「录制中」语义随之悬空——见 D-5）。
2. **断网**（页面侧 `es.close()` 模拟，注明手法）：2.5s 静默看门狗判 `lost`；**LINE 灯断链即熄执法确认**（`#pilot --lit` 1→0）。
3. **杀 serve**：**E5 死相设计良好且真实工作**——SIGNAL LOST 丝印＋缓跳红 beacon 显形、机器整体冷化（brightness 0.82/saturate 0.72）、床音继续（0.0077，「机器转冷不断气」）。截图存证（会话记录中；复现：起任一 serve→开页→手势→杀 serve→6s）。
4. **附赠·第四式（意外撞出）**：「先开机后开工」——空母带房起 serve→开页→会话文件后至并生长，页面 SIGNAL LOST **固死**（serve 侧热、刷新即愈，页面无「从无到有」的 live 重协商）。真实时序：用户先跑 foley 再开始干活。复现：`FOLEY_PROJECTS=<空目录> node cli/index.ts <port>` → 开页 → 往目录里铺会话文件。

## 二、缺陷与割裂清单（按严重度排序）

> 标签：【实证】可复现/有数值；【推断】机理成立待复核；【口味】候船长肉眼耳朵。

**D-1【实证】产品处于「不可获得」态，README 与实况互为谎言。**
`npx foley` E404；README 又已按发布态预写（Quickstart 第一行即 npx）。GATE 全部候签项停在 2026-07-07 的红队/船长签收，此后五周的巨量工程（体验期五 P0、视觉重铸、高板接线）没有进入任何发布流程。**咬人场景：今天任何一个被 hero.gif 吸引的人，第一条命令就死。**（发布本身有人类扳机与镜像纪律在册，此条不是叫嚣「立刻发」，是把「未发布」定性为最高严重度缺陷入册——它让其余一切缺陷都失去外部判据。）

**D-2【实证＋口味】素面首帧＝「褪色白板上的悬浮器件」，暗房缺席，开关不可见。**
新开页（pre-gesture）四次复现：面板呈灰白（压暗仅 `brightness(0.66)` 乘在**亮色渲染板**上）、双盘读作黑洞、纸窗黑块（`--paper-idle`＋canvas 截图失真叠加）、REC 一粒红、**选择器旋钮融入白板几乎不可辨**——第一眼没有「黑暗画布里浮出的机器」，也没有可见的「门把手」。琥珀宪法序言「黑暗是画布，面板从黑暗中浮出」在**用户看到的第一帧**上不成立。通电后（暖光暗房）质感与素面判若两机——反差本身证明素面是欠费状态而非设计。候船长肉眼终审，但结构性事实（压暗系数/可见性）是实测的。

**D-3【实证】「点空白处」一击三事：声桥解锁＋快拧 POST＋架沿下摇并发。**
页面下部空白（新手最自然的「点一下试试」位置）落在架沿下摇命中区：一次点击同时通电、开声、**镜头沉入半层**。两页独立重现；顶部点击则干净。新手的第一击有相当概率把自己扔进带库中间层，且不知道发生了什么、更不知道怎么回来（D-6）。

**D-4【实证】DUB 示能承诺与兑现割裂——「死活不明 2.0」。**
`#dub-key` 呈 `dub-ready` 咬合芒（承诺「剪得出东西」）→按下→`latched`→约 13 秒后**静默回 idle**。「无戏可剪」的摇头是一次性短动画，不注视按键即错过，事后零残留。P0-2 丙.3 修掉了「不可用时静默」，但「可用态按下最终无果」走回了静默。物理对照：售货机拒币会把硬币退回槽里（持续证据），不是抖一下完事。**连读完全部史料的审计员都没能完成一次 DUB**——普通用户首周 DUB 完成率可以预判为个位数百分比。

**D-5【实证＋推断】「在录吗」没有单一事实源——尸体上的 rec-live。**
三个真值源并存：服务端 transport 广播（驱动 `body.rec-live`→REC 红呼吸）、页内 `liveActive`、live 连接状态机（connecting/live/lost/gone）。实测：serve 被杀、SIGNAL LOST 已亮，`rec-live` 仍挂在 body 上（无人撤）；替身死亡 3 分钟，REC 照红。顶段视界下「REC 呼吸＋失联红点」可同屏。旋钮 OFF（素面）与 REC 红同帧也是同根。**REC 是全机唯一合法信号红（修宪甲原文），它的语义必须像红灯一样可信。**

**D-6【实证】镜头导航回程失灵。**
下摇后 Escape 不回位（towerY 纹丝不动，实测）；滚轮回程在本环境工具层超时未能定谳（存疑不定罪，候真手滚轮）。若两条回程都断，掉层用户唯一出路是刷新。另：两次读数 towerY 从 -457.72 漂到 -632.45（40 分钟无交互）——两种解释（点击缓动的中途值 vs 终点值；或慢漂无界累积）本审计无法分辨，**「镜头导航最长停留会怎样」没有编舞值班答案**（值班律适用而未答）。

**D-7【实证】「先开机后开工」固死失联（附赠第四式）。**
死相册 §1.3.4。刷新即愈，但新手不知道要刷新——他看到的是一台亮着 SIGNAL LOST 的新机器，而他的 agent 明明在干活。

**D-8【实证】回归资产失信进行时。**
①`audit/p0-2-transport/repro/transport.mjs` 腐烂假红：spy 钩 2026-07-08 的 `sound.pause()` 旧链，声批刀二改链（`bridge.pause→E.paused`）后未随迁——行为正确、脚本判死，谁跑谁拿红，回归资产公信力在漏气。②selector/counter 状态机（含关机三档全向转移表）零金测（渲染批复盘挂账自认）。③金测在本机 174/174 全绿但账本口径是 171/174（B4 三例环境性）——「哪些红是真红」需要一张活的对照表，否则每个新终端都要重新踩一遍。

**D-9【实证＋推断】纸带无时间锚，「第二天早晨」残值极低。**
live 纸窗约 45 秒（PAPER_SPEED 16.5）；纸上有格线无标尺、接带痕表达「有空洞」不表达「多久」。卡片 card.png 同病（见 D-10）。用户睡醒后，昨夜的风暴只剩卡房里一条无字曲线。「纸带即时间轴」是宪法级隐喻，但**这条时间轴上读不出时间**。

**D-10【实证＋口味】卡片是漂亮的哑巴——社交货币成色实证性低。**
真卡 card.png 实查：奶白纸理＋牛血红曲线＋接带痕＋FOLEY 水印，工艺成立；**零文字、零身份、零战报**。安静会话（多数会话）的卡≈一条直线。旁观者读不出「谁、干了多久、经历了什么、结局如何」中的任何一项。船长「无分享欲」判语有了实物证据。（011-R2/设计二的章、盒面、盘径阶梯立法正是解药，冻结中——见机会 O-3 的最小刀。）

**D-11【实证＋口味】翻牌把「机器最大的字位」给了背景音乐。**
live 模式翻牌揭 SATURATION（画外唱机曲名，设计二法理成立），但它是页面最大最亮的文字，新人必然误读为「我的会话叫 SATURATION」。会话身份（真实仓名/开场白）只存在于货架 title 悬停与档案行小字。**语义权重与视觉权重倒挂。**

**D-12【推断】「思考型会话」在机器上＝无事发生。**
蒸馏按 content-block 行工作（粒度实证正确），thinking/text 块不成事件——一个苦思冥想 5 分钟的 agent 与一个死掉的 agent 在盘面上同相（盘停、床近隐）。证据律（「引擎只算证据」）之内合法，但用户「我看它明明在转圈思考」的体感落差真实存在。多 agent/长思考工作流普及会放大此感。

**D-13【口味】三语割裂现状复述**：CLI 中文、面板无字（英文器件铭牌＋SIGNAL LOST 英文状态句）、README 英文。known-limit 在册；夜审补充一问：**CLI 的中文是「暂时没翻」还是「身份」？**若是身份，README 应该骄傲地说出来而不是道歉式标注。

**D-14【实证·小】** doctor 的 serve 探测只查默认 4173，起在别处即报「没起」（自身有诚实标注，但「它到底接了啥」的一条命令承诺打了折——doctor 明明知道怎么找端口参数）。

**O-5 冰箱清点（手令指定问）。**

**O-7 手令终极二问。**

## 五、宪法挑战席记录

> 规矩：先钢人后开火；打不穿钢人的不入册。

**C-1 挑战「主画面无数字」的边界解释（打的是执行域，不是法本身）。**
钢人（最强形式）：数字即量化，量化即焦虑；这台机器是仪器不是仪表盘，器件呈值与方差、两义封顶；数字活在计数轮里，镜头凑近才入画——克制是产品的脊柱，一旦面板出现「47%」这台机器就死了。
开火：完全接受面板域禁令；但当前实践把禁令**溢出**到了档案域——纸带无标尺（D-9）、卡片无战报（D-10）、连「1 格 = 多久」都不可知。而 REPORT.md 满篇数字、卡目录里 curve.csv 全是数——**档案域早已有数字，只是没有给人看的数字**。法理裁量申请：明文划界「面板域禁数字（现行法不动）；纸/卡/册为档案域，档案数字合法且必要」。代价：需要一次字法批（蚀刻/打字机体的档案字规范），且守门人要盯住「档案字向面板爬」的走私。判据：D-9/D-10 两病灶在不改一件面板器件的前提下痊愈。
**判定：钢人立住了，法不动；打穿的是「法域边界未成文」这块空白。建议入宪一句边界条款。**

**C-2 挑战「交互法：界面味交互不上面板」之「可发现性零补偿」。**
钢人：机器的主人对机器做机器的动作（凑近/撕纸/轻敲），按钮菜单是屏幕的语言，一旦上面板，七十年代的诚实全毁；dev 抽屉已是唯一豁免。
开火：法条只禁止了「界面味」，没有回答「主人怎么知道机器接受哪些动作」。实证：DUB 的「提议→撕纸」链路，一个读完全部正典的审计员都没有走通（D-4）；三事并发（D-3）说明「哪里可点、点了会怎样」全靠碰。物理机器有其自己的教学语言——**新机器出厂附「操作卡」**（一张印刷品，不是 UI）：Nagra 的面板下就贴着操作简图。替代案：①首启一次性「纸质操作卡」（一张可关闭的印刷风格卡片，蚀刻/铅印语言，介绍旋钮/DUB/带库三动作，档案域合法）；②latched 等待期给键一个持续「咬住」态而非瞬时摇头。代价：一件新纸品的设计工作＋「首启打扰」的克制拿捏。判据：新用户首周 DUB 完成率从预判的个位数 % 上到两位数。
**判定：钢人立住，法不动；打穿的是「无字＝无教学」的默认等式。教学可以是纸做的。**

**C-3 撤回记录（打不穿，如实入册）：** 曾试挑战「机器永恒，介质易朽」（论点：永恒的机器积累不了「我的机器」的依恋）——钢人反杀：依恋应由介质承载（带库、纸卷、章），机器崭新恰是「时间只属于内容」的立场本身。挑战不成立，收回。
