# TAPE-0 感官设计白皮书 v1 —— 床与前景

> 地位：v1 音画层的宪法级文件，与《琥珀宪法》（视觉）平级互补；本件以声音为主、以 §5 绑定表连通视觉。
> 一级输入：EAR_REPORT 证词 F1–F5（船长）；既有教义（血氧仪×Eno、三层信号、保真度即张力、磁带宇宙、诚实条款）。
> 总纲出处注明：**"常态沉入 lofi 背景床，只让偏差冒头"——首听者证词，F4/F5 之收敛，本文一切设计由此展开。**

---

## 1. 总纲：一张床，几件事

v0 的病（船长亲耳诊断）：声音只有离散事件，曲线只有连续张力，两轴正交、互不解释。v1 的方子：**双层架构**。

- **连续层（床）**＝状态的身体。永远在响，编码 T / A / wow / phase / weather。它是"曲线的声音"。
- **离散层（前景）**＝事件的标点。拨弦、咔哒、和弦、跳针。它是"时刻的声音"。
- 三层信号旧教义映射：背景态＋天气态 → 床；事件与呼唤 → 前景。呼唤词汇 ≤4 铁律不变。
- 诚实条款延伸到响度：**床的能量包络必须诚实追随 T**——这是 F5 的直接回应，且机器可验（§6）。

## 2. 连续层（床）规格

### 2.1 结构：四条 stem，垂直分层（游戏音频正统手艺）
| stem | 内容 | 门控/驱动 |
|---|---|---|
| S1 基底 | 温暖 pad＋磁带底噪＋房间感（Eno 式互质长度循环，织体永不精确重复） | 永远在（IDLE 时唯余此层最弱态） |
| S2 律动 | 慵懒 boom-bap 骨架（拍点稀疏，力度轻） | A > 0.3 淡入；密度 ∝ A |
| S3 张力 | 暗色弦乐/低音 pad，和声悬挂音比例 ∝ T | T > 0.35 渐入 |
| S4 保真总线 | **磁带老化处理挂在整体输出上**：hiss 电平、wow&flutter 深度、高频衰减、偶发 dropout | T 与 wow 驱动（见 2.2） |

### 2.2 映射表（起手值，全部进 sound-params.json，可调）
- **T → 和声张力**：sus/add 音概率 = T；**T → 滤波**：主滤波截止 8kHz→1.8kHz 随 T 线性下压；**T → 磁带磨损**：hiss −60dB→−38dB，wow 深度 3→22 音分，高频搁架 0→−6dB。
- **A → 律动密度**：音符触发概率 0.2→0.9；tempo 恒定 **72 BPM**（节拍是地基，永不漂）。
- **wow → 走带不稳**：pitch LFO 速率/深度（与 S4 的 wow 合流；"稳定地烂"= T 高 wow 低 = 音暗但走带笔直；"反复横跳"= wow 高 = 音准微醺）。
- **phase**：IDLE = 仅 S1 最弱态＋星河静场；WAITING = 床转**半终止悬停**（属和声延音，与 ASK 动机同族——整张床替琥珀管一起呼吸）；DONE = 正格终止 → **真静默 ≥4s**（洗碗机时刻，静默即通知）。
- **weather → 场景**：档位切换只在**小节边界**执行（既有教义）；下行切换（雨→多云→晴）即"消散的表达"——**风暴的呼气不是一个音符，是滤波张开、hiss 退潮、磨损复原的那三十秒**。此为"消散≠解决"纪律在声音层的兑现：被遗弃的卡碟无号角，但雨停有耳可闻。
- **调性**：每仓库一调（repoKey = hash(repo) → 五声音阶主音与色彩）——主力项目永远"听起来像回家"。
- **平滑纪律**：一切连续参数过 slew limiter（时间常数 250–800ms 分参数定），禁跳变；参数更新对齐到 1/8 拍网格。

## 3. 离散层（前景）规格

### 3.1 词汇表（≤6 乐音 ＋ 呼唤 3，超编即删）
| 事件 | 声音 | 备注 |
|---|---|---|
| WRITE-OK | 拨弦（Karplus 类） | 音高＝五声音阶内由 **targetHash 选音** → 同一文件反复出现同一动机（**文件的主题曲**，免费的叙事性）；力度/亮度 ∝ 当刻 T（F1：强度进声音） |
| WRITE/READ-FAIL | 低音区闷拨弦 | 保留 v0 的"成功高、失败低"音区分裂——盲听证明它承载了信息 |
| READ-OK | 纸页翻动（极轻，最先被习惯化沉床） | foley 物理触感 |
| RUN-OK | 打字机回车铃（轻） | 带 test tag 且触发 RESOLVE 时让位给和弦 |
| SAVE-OK | 卡座按键咔哒＋低音锚 | S≥saveResolveMinS 才附和弦（M1.8-F 裁决） |
| SPAWN | 新声部淡入一小节 | 多轨预演 |
| **RESOLVE** | 和弦解决（正格进行，音色待雕，F2） | 呼唤级，直通不量化 |
| **STUCK** | 跳针（同槽卡碟专属） | 呼唤级，直通；同 sig 一响，CLEARED 前不复鸣 |
| **ASK** | 半终止动机（温柔穿透，频谱专区 2–4kHz） | 呼唤级，直通；礼貌性重复（90s 一次，音量不升级） |

量化：乐音级前景量化到 1/8 网格（宁迟勿早）；呼唤级三种永不量化。

### 3.2 告警经济学（F4 的机械实现：习惯化曲线）
- 同类前景音在滚动 60s 窗内第 n 次出现：音量 ×0.85^(n−1)，下限＝沉入床的响度（成为织体，不消失）。
- **新颖即重置**：换类事件、或该类静默 >60s，衰减归零重来。
- **豁免**：呼唤级（RESOLVE/STUCK/ASK）与 DONE **永不习惯化**——报警的完整性高于一切。
- 效果自查：顺风午后＝床＋偶尔一声清晰拨弦；风暴＝床变暗变糙、低音拨弦沉成织体、跳针刺出——正是"新手不再被每一声拽走"（F4）与"突变仍令人警觉"（F3 保护条款）的并存。

## 4. 呼唤态纪律（重申，一字未松）
词汇 ≤4（RESOLVE / STUCK / ASK / DONE-静默）；音量不是报警维度——靠音色新颖、前置微静默、频谱专区穿透；永不习惯化；永不量化。

## 6. 验收（v1 声音层）
1. **机器判**：五带回放，床响度包络（1s RMS）与 T 曲线 Pearson r ≥ 0.6（storm 带必测）——F5 的可执行化。
2. **盲听 v2**：单卷冷听（**不许对比**）30–60s 判日子类型 ≥4/5——F1 修复的验收；G2"愿意陪一下午"≥"看情况"。
3. **F3 保护性检查**：突变警觉仍在（升频/骤变时"想回头看"的证词必须复现）。
4. 呼唤级三音在床最响时仍一耳可辨（频谱专区实测）。
5. 阻断级听感条目 = 0。

## 7. 实现注记
- 响度纪律（起手值）：床 −26 LUFS 邻域浮动 ±4（随 T）；前景峰 −18；呼唤级可至 −14；总线真峰 ≤ −1dBTP。
- **调音抽屉**：`?tuner=1` 开隐藏参数面板（仅 dev），船长边听边拧——耳朵驱动的扫参。sound-params.json 与引擎 params 同级治理（hash 上报）。
- 资产路线：v1.0 全合成（Web Audio 原生已证足）；真 foley 采样（铅笔/翻页/卡座实录）为 v1.x 升级项，入冰箱勿抢跑。库选型：如需，Tone.js 以 vendor 构建期依赖进 stage/（运行时零网络不破）。
- 一切"感知常量"（响度、频谱专区、习惯化系数）目前标定于 n=1 语料——已知界限，公测后以社区耳朵复校（冰箱：开箱调音）。
