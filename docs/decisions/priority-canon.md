# 优先级正典（M1.7 §5 确立；SPEC 顶部横幅 & verdict 头注同源）

**安全硬禁 > 协议冻结 > 判据 > 施工令实现细则 > 库偏好**

- **安全硬禁**：零网络、零遥测、隐私膜（原始 JSONL 只在蒸馏器读一次；工具输入/输出正文/对话永不落盘）、redact 红线——有牙的闸门是 `golden/privacy.redteam.test.ts`，与金测试同跑永久在册（M1.8-F④）。
- **协议冻结**：`protocol/index.ts` schema v1 只增不改，改动需架构师签核。
- **判据**：verdict.json（判据生命周期 status 三态：active / informational / retired；修订需提交信息签核行）。
- **施工令实现细则**：与上位冲突时停手写 FEEDBACK，不先斩。
- **库偏好**：最末位——例：SPEC §7 曾指名 Tone.js，因"零网络+零运行时依赖"硬禁而以原生 Web Audio 实现同语义（v0 探针现实修正）。

## 审计脚注（NIGHT-1 质疑席，留作镜鉴）
审计曾指出："协议冻结被逐字节严守，而安全面的 redact 明文残留无人守、无金测试拦——若安全硬禁真在正典顶端，隐私红线该比 schema 洁癖得到更多机械保护。" M1.8-F④ 以常驻隐私闸门回应；此教训入册：**正典的次序要用机械守护的强度来证明，不是用嘴。**
