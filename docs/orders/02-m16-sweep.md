## 2. 引擎修订（教义级，随本轮实施）

### 2.1 RESOLVE 多态化（修复"测试驱动世界观"偏见）
"解决"的三种形态，任一发生即发射 `RESOLVE` 时刻并做乘法泄能：
1. **测试转绿**：带 test 标签的 RUN-OK 且 S > `release.testResolveMinS` → `S ×= 0.6`（原规则不变）；
2. **提交**：SAVE-OK → `S ×= 0.5`，**现在同时发射 RESOLVE**（原先只泄能不发时刻）；
3. **卡碟打破**：由同 verb+tool 的 OK 引发的 `STUCK_CLEARED`，且当时 S > `release.jamBreakMinS` → `S ×= release.jamBreakFactor` 并发射 RESOLVE。
**语义纪律**：窗口过期引发的 STUCK_CLEARED **不**触发 RESOLVE、不泄能——消散不是解决。引擎内部记录 clearedBy（ok / expiry），报告分列计数（协议 schema 不加字段）。
新参数：`release.jamBreakFactor = 0.7`，`release.jamBreakMinS = 0.25`。

### 2.2 wow v2（Nagios 借骨）
wow 从"失败率 EMA"改为**成败交替率**：取最近 20 个有结果事件（OK/FAIL 序列），统计相邻跳变（OK→FAIL 或 FAIL→OK），按近因加权（最旧 0.5 → 最新 1.5，线性），跳变加权和 / 最大可能加权和 = 原始 wow，再过 30s 平滑。语义分工从此正交：**T 管"多糟"，wow 管"多飘"**——稳定地烂（高 T 低 wow）与反复横跳（高 wow）是两种不同的病。
参数：`companions.wowWindow = 20`（含义变更：改为"最近 N 个有结果事件"）。

### 2.3 教义一条：改物理，不改尺子
`weather.up` 阈值（0.25/0.50/0.75）与迟滞是**感知常量**，是尺子；校准永远只动力学参数（权重/幅度/τ/repCap/泄能），**永不**为凑指标动尺子。任何情况下不得将天气阈值列入扫参维度。

## 4. 扫参器 `cli sweep`（本轮核心交付）

### 4.1 网格（`sweep.json`，共 4×3×3×2×2 = 144 组）
| 维度 | 取值 |
|---|---|
| `stress.verbWeights.READ` | 0.15 / 0.18 / 0.21 / 0.24 |
| `amplitude.failDefault` | 0.3 / 0.4 / 0.5 |
| `decay.tauActiveSec` | 120 / 150 / 180 |
| `stress.repCap` | 4 / 6 |
| `release.testResolveMinS` | 0.2 / 0.3 |

其余参数固定为现值（含 §2 新参数）。每组 × 五带全量回放。引擎确定性保证下，sweep 本身也须确定：同网格两跑 → 结果 CSV 逐字节一致（金测试）。

### 4.2 产物
- `runs/sweep-<ts>/sweep_results.csv`：每行 = 组合参数 + 五带全部指标实测 + 各带 PASS/FAIL + 全绿标志。
- `SWEEP_REPORT.md`：全绿组合数；**领奖台**（按 §4.3 规则排序的前 3 名，含五带关键指标）；冠军 params.json 全文与一行入选理由。

### 4.3 冠军规则（确定性，禁自由裁量）
在**全绿组合**中依次择优：
1. 距现行参数的归一化步数之和最小（改动最少者胜）；
2. 并列则 storm 峰值 T 最低者胜（留头顶余量）；
3. 再并列则 τ 最小者胜（雨过天青更快）。
冠军即新 params.json，提交并更新 hash。
**零全绿时**：不选冠军、不改参数，输出**帕累托前沿**（恰好违反一条判据的全部组合，注明各自违反项与差距），停在闸门等架构师裁决——判据是假设，扫参是实验，实验推翻假设是合法结局。
