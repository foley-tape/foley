## 2. 风暴换尺：雨量 R（burn-rate 同构）

- 定义：`R = ∫ max(0, T − 0.50) dt`，按活跃时间积分，单位 T·min；分 episode 与全带各报一次；**五（六）带全部计算并记录**，不限风暴。
- 判据修订（写入 verdict.json）：
  - storm / storm-b：峰值带不变 **且**（RAIN+STORM 占空 ∈ [15%,45%] **或** R ≥ 2.0）。
  - busy 增加特异度护栏：R ≤ 0.3。
  - 阈值标注 provisional：双尺并行记录一轮，下轮以数据裁定去留。
- 依据备注：阵发型风暴会从"持续时长"类判据的指缝漏光（快窗口逮急性、慢/积分逮累积，二者合并是行业成熟形态）；雨量对尖峰面积照算，天然对阵发公平。

## 3. targetHash（卡碟误清修复，蒸馏带升版 distill/2）

- 蒸馏事件新增 `targetHash` 字段：RUN 取规范化命令头（前 2 个 token、抹路径、过 ≥16 位 token 净化）后哈希；READ/WRITE 取主目标路径哈希；其余动词可空。
- **卡碟清除条件收紧**：同 verb + tool + targetHash 的 OK 才算 ok 型清除；充能侧 sig 规则不变。
- 全部磁带按 distill/2 重蒸；金测试：bash-A 卡碟不被 bash-B 成功误清。
