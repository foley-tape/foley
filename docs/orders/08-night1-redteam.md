# TAPE-0 夜班令 · NIGHT-1 对抗式审查（封版前红队）

> 阅读对象：**全新会话**的施工方（旧会话勿用——本次要的就是没被教义浸泡过的眼睛）。
> 性质：**只读审计**。除 `audit/` 目录与复现测试外，禁止修改任何源码、参数、判据、磁带。发现 ≠ 修复；修复权在架构师晨间裁决之后。
> 开工前置：`git status` 必须干净；切分支 `audit/night1`；全程无网络（既有硬禁）。

## 0. 阶段零 · 盲答（必须最先做，读任何项目文档之前）

只读本文件，**不看** SPEC/ORDER/代码，先写 `audit/PREREG_DESIGN.md`，回答：
"给你一条 coding agent 的会话日志流，要把它实时翻译成一根张力指针和极少数几种声音——你会怎么设计？事件怎么筛？张力怎么算？怎么防误报？怎么表达'解脱'？"
写完（约 30–60 分钟深度即可）再开卷读全仓。之后所有与现设计的**分歧点**逐条记入报告"分歧席"——分歧不自动等于缺陷，但每条都是免费的第二视角。

## 1. 四支红队（每队附一条预埋气味线索，顺藤深挖，不限于此）

### 红队A · 隐私攻击者
目标：证伪"蒸馏带接近可分享、无原文泄露"这一未来公开主张。
- 构造对抗性原始 JSONL（伪造样例，不用真带）：把"密钥"藏进错误首行、工具名、URL、命令参数、超长行、非 UTF-8、emoji 混排——看什么能活着穿过 distill/2 与 `--redact`。
- **气味线索**：errClass 的 ≥16 位 token 净化抓不住**短密钥**——10 位口令、8 位 PIN、`-pSECRET` 式内嵌参数会否存活？归一化（抹路径/数字/hex）的绕过面有多大？
- 时间戳/episode 结构是否构成可指纹化信息？

### 红队B · 数学审计员
目标：让引擎在边界上出丑。
- 乱序时间戳、时钟回拨、重复 seq、跨午夜/时区、零时长 episode、事件断流后巨大 dt 一次性灌入（弹簧离散积分会不会爆振？）、S 极大时的浮点行为、wow 在样本 <2 时、迟滞恰好落在阈值等号上、repWindow 边界差一毫秒。
- **气味线索**：确定性主张跨 Node 版本/平台是否真成立（浮点累加顺序、`Math.exp` 一致性）？写一个双跑比对脚本验证。

### 红队C · 恶意磁带工程师
目标：让 adapter 崩、慢、或算错。
- 截断行、单行 100MB、tool_use 无 result / result 无 use、嵌套怪胎、未来时间戳、负时间戳、同毫秒千事件。
- **气味线索**：tag/save 正则跑在用户可控字符串上——审计 ReDoS 面与误匹配面（命令里带 "test" 一词但不是测试，会被贴 test 标签吗？后果链到 RESOLVE）。
- 性能：合成 10×、100× 体量磁带，测 replay 时间与内存曲线，报拐点。

### 红队D · 合规审计员
目标：文档-代码-测试三方对账。
- 每条施工令条款 → 代码落点核对；未记录在 FEEDBACK"现实修正"里的静默偏离 = 发现。
- **气味线索**：23 条金测试逐条质检——它们各自**真的**在断言有意义的行为吗？有没有同义反复、断言过弱、或测试自己 mock 掉了被测物的？
- verdict/2、params、蒸馏带三者的 hash 链条是否闭合可追溯。

## 2. 通宵耐力测试（夜的专属工序）

- 写 `audit/soak/`：合成会话发生器（随机但可复现种子）以真实节奏（每 5–40s 一事件，偶发风暴簇）滴写监听文件 **6–8 小时**；`cli live` 挂机。
- 采样进程 RSS/CPU 每分钟一记；MomentEvent 发射时刻 vs 理论时刻的漂移分布；过期型 CLEARED 的 tick 对齐是否随时长退化。
- **稳健性要求**：harness 自记录、自终止、与审计会话解耦（后台脱离进程 + 滚动日志 + 独立 `soak-summarize` 脚本）——即使本会话先结束，晨间任何人跑一条命令即可出 `SOAK_REPORT.md`。

## 3. 产出与纪律

- `audit/AUDIT_REPORT.md`：每条发现 = 严重度 {P0 阻断封版 / P1 封版前应修 / P2 v1 处理 / P3 冰箱} ＋ 复现步骤 ＋ 证据（可附失败测试于 `audit/repro/`）＋ 建议修法 ＋ 一句"为何现有金测试没抓住"。
- 卷首"**十大发现**"执行摘要，按严重度排序；灌水式风格意见一律 P3 折叠。
- "**分歧席**"：阶段零盲答与现设计的分歧清单（不评对错，供架构师读）。
- "**质疑架构师席**"：对判据、教义、优先级正典本身的质疑，显式欢迎，单列不修。
- 晨间流程：架构师裁 P0/P1 → 修复令 → 修完重跑五带＋盲听结论合流 → v0 封版三件套照旧。
