# FOLEY 设计案 · 剪辑带（DUB）—— 预告片轮

> 性质：架构师设计案（与《感官设计白皮书》《琥珀宪法》同级，画声之外的第三件：**剪辑正典**）。
> 施工归属：Track-STAGE 全责；`sound/` 只读消费（写权仍在 Track-SOUND，API 缺口走提请）；cli 命令暂缓，先以 `stage/tools/dub.mjs` 承担自动化（围栏内），正式 `foley dub` 候总务轮迁移。
> 素材诚实条款全程适用：一切演示接带回放真实磁带，dub 元数据记录选段窗口与磁带哈希，无从摆拍。

---

## 0. 灵魂

**预告片不是视频剪辑，是一盘用剃刀剪出来的接带。**磁带时代的剪辑=刀片＋接带纸；我们的剪切点=接带痕＋接带音。导出物是一盘 dub（对倍复制带）：一段 mp4、一条撕下来的纸（选段墨迹 PNG，即分享封面）。纸边按真实记录纸做派印极淡的 `FOLEY` 边字——每条流出的接带都替机器署名，此外无水印。

三条铁律：
1. **机器提议，人来撕**——自动选段以齿孔虚线呈现，撕开才成立；机器永不擅自导出。
2. **段内恒速，换速只在接带处**——磁带机不会平滑变速；时间扭曲的蒙太奇由"每段一个恒定倍速"拼成。
3. **渲染离线且同源**——画与（未来的）声由同一份 cuts 时刻表驱动，导出是回放的另一种消费者，不是屏幕录像。

## 1. 戏剧计分器（选段引擎，纯函数，`stage/js/cut.js`）

- **戏剧密度**（1s 分箱）：`d(t) = a·T + b·|ΔT|/Δt + c·momentRate(60s窗)`＋事件加成：STUCK 边沿 +0.5、RESOLVE +0.6、ASK 边沿 +0.2、天气上行切换 +0.3、DONE +0.4。系数入 `cut-params.json`（起手 a=1.0 b=0.8 c=0.3，治理同 params：hash 上报、改动记录）。
- **叙事文法**（不是 top-N，是一条弧）：
  - `[OPEN]` 会话起步或首次活动，2–4s，8× 速；
  - `[RAMP]` 张力爬坡代表段，4–8s，2–4×；
  - `[PEAK]` 全带 ∫d 最大的连续窗（Kleinberg 突发权重精神：强度×时长），8–15s，**1×**——高潮必须原速；
  - `[TURN]`（可选）峰外最大单次 |ΔT| 时刻，3–6s，1–2×；
  - `[CLOSE]` DONE／末段消散，3–6s，1–2×；正格终止感优先（若带内无 DONE，取最终衰减段）。
  - 桥段 1–3 条（8–16×，纸带飞驰的延时感）连接锚段。
- **时长求解器**：目标 45s（预设 30/45/60/90），按锚段优先级分配，桥段吸收余量。
- **确定性金测试**：同带＋同 cut-params → cuts.json 逐字节一致。
- **影子指标**（informational 首轮，先验影子自身）：选段覆盖率 = ∫d(选中)/∫d(全带) ≥ 0.6，且选中时长占比 ≤ 全带活跃时长的 15%（长带口径；短带放宽）。
- 输出：`cuts.json { tapeHash, segments:[{t0,t1,speed,role}], version, paramsHash }`。

## 2. 交互（交互法全程合规：按键、齿孔、撕）

1. **DUB 键**上面板（七十年代卡座对倍复制的正统按键；字法丝印 `DUB`；器件法注：DUB 是操作件不是仪表，不承载读数）。
2. **按下 DUB = 机器提议**：走纸上沿 cuts 边界打出**齿孔虚线**（rouletting），随后卷轴快速穿带做一遍预览（选中段闪过，落速经过 PEAK）；十秒内无动作则齿孔缓缓淡去（提议可拒绝，机器不催）。
3. **确认 = 撕**：顺齿孔按住拖动，纸沿孔分离、边缘毛化、纸条翘起（220ms 软惯性，同 loupe 镜头语言）→ 进入渲染。**手动模式**：不按 DUB 直接在纸上按住拖选任意区间，齿孔跟手，撕即导出该段原速剪。
4. **渲染进行时**：双轴快速穿带、计数轮飞转、纸位标traverse——进度由机器的动作陈述，无进度条、无数字（精确数字照旧只活在 HUD）。
5. **产物呈现**：撕下的纸条滑落到胡桃木台面上歇着——纸条即选段墨迹 PNG（含接带痕、齿孔边、FOLEY 边字）；旁边一枚 mp4 卡。点击落盘（浏览器下载 ＋ `runs/dubs/foley-dub-<tape>-<date>.mp4|.png` ＋ `dub.meta.json`）。
6. **时长选择无数字**：预设以纸长呈现（齿孔间距即时长）；精确秒数在 HUD。

## 3. 渲染管线（离线、确定调度、快于实时）

- **帧渲染**：以 cuts 时刻表驱动回放管线逐帧步进（固定 30fps 输出网格；段内恒速），OffscreenCanvas 或隐藏画布；**每帧渲染后必须等一次 rAF 再捕帧**（GPU 合成完成前取帧会得到陈旧缓冲——已知陷阱，写入实现注释）。镜头层动态颗粒保留（导出介质适配法：mp4 走高码率）。
- **编码**：WebCodecs `VideoEncoder`（探测 `isConfigSupported` 后选串：H.264 `avc1.4d0034` 优先）→ **Mediabunny 或 mp4-muxer**（MIT，构建期 vendor 入 `stage/vendor/`，登记 LICENSES.md）封装标准 MP4。兜底链：AVC 不可用 → VP9+webm（webm-muxer）；音频编码器缺席（Safari<26 只有视频接口）→ 视频无声导出并注明。硬编加持下目标 **≥2× 实时**（影子指标）。
- **码率预设**：1080p30 约 10–14 Mbps（动态颗粒吃码率，宁高勿糊）；720p 预设减半。GIF 次级出口：纯 JS 编码器、静态颗粒单帧化法、≤8s、720p 内。
- **音轨（M-T3，候声音过耳）**：`sound/` 离线渲染器按同一份 cuts 时刻表出 PCM（同源同钟的导出延伸）→ AAC（可用则 mp4 内封）或 Opus（webm）；接带处叠**接带音**（刀片/胶带 foley，CC0 一枚入资产清单）；结尾强制正格终止＋≥2s 静默。AV 同步影子：≤1 帧。sound/ 若缺"按时刻表渲染"钩子 → 提请 Track-SOUND 增补（预计一个函数签名的量）。
- **确定性诚实条款**：cuts.json 与音频调度严格确定（金测试）；像素因 GPU 渲染不保证逐字节跨机一致——如实标注，不伪称帧哈希确定性。
- **隐私**：渲染只消费蒸馏字段；歌词通道未启用，画面永无会话文本；dub.meta.json 只含窗口/哈希/预设。

## 4. 里程碑

| 里程碑 | 内容 | 交付 |
|---|---|---|
| **M-T1 剪刀与纸** | 计分器＋cuts.json 金测试＋DUB 键＋齿孔提议＋撕确认＋纸条 PNG＋dub 元数据 | 撕纸仪式屏录一段；五带自动选段的纸条 PNG 各一（素材诚实）；覆盖率影子首采 |
| **M-T2 复制机** | 离线帧渲染＋WebCodecs mp4（无声）＋预设＋边字＋海报帧（PEAK 取帧） | 真接带 mp4 三支（storm／jam／一支日带）；渲染速度影子 |
| **M-T3 配乐接入** | **门：声音相过耳后开工**。音轨渲染＋接带音＋终止式收尾＋AV 同步影子 | 有声接带一支＋同步实测 |

M-T1/T2 与声音手术完全解耦——**预告片不等声音**；T3 是声音痊愈后的合龙。

## 5. 验收

1. cuts 确定性金测试绿；覆盖率与渲染速度影子首轮采数（informational）。
2. **妈妈测试·接带版**：一个不了解项目的人看完 45 秒 storm 接带，能说出"这台机器经历了一场风暴，最后停机了"——弧线无需解说自明。
3. 交互法自查：全部交互 ∈ {按键、拖选、撕}；面板零数字维持。
4. 素材诚实：三支演示接带的 dub.meta.json 随交付归档。
