## 0. 本令裁决（全轨知悉）

1. **唱片血统条款（入宪）**：内置唱片必须人类制造；每首 vendored 曲目配 PROVENANCE 三件套（来源链接／许可证快照／作者身份）。AI 生成目录（open-lofi 等）不入厂机，入**淘碟指南**页并诚实标注"AI 生成（Suno v5）·CC0"。用户唱片架（~/.foley/records/）内容属用户自治。
2. **零静默网络（红线修订）**：机器永不自行联网；唯一例外＝首启**明示征询**的出厂唱片下载（展示目标 URL／体积／SHA-256，同意才取，验哈希落盘；拒绝则房间层运行并提示唱片架路径）。README 隐私段同步更新此句（发布前冷读者复核）。
3. **npm 减重令**：唱片不进 tarball；音频资产挂 GitHub Releases；`npx foley` 冷启动目标秒级。
4. **委托首版唱片（发布叙事资产）**：船长亲自发起——预算几十美金级、目标 1–2 首、买断或明确授权、征得艺术家同意讲述合作故事。不阻塞任何工程轨。
5. **重媒体归档策**：追认 L-1＋L-2；原则＝**仓库存指纹不存重媒体**（meta＋海报帧入库，视频迁 Releases）；LFS 迁移在多会话并发下否决。公开镜像策（发布日以干净历史开公开镜像、私库存全史）列发布物料轮案头。
6. **安全组合拳**：追认选项书 X-1＋W-1＋W-3。原则约束（凌驾于方案标签）：①一切 URL 参数未经白名单校验/转义不得入 DOM 或文件路径；②serve 只绑 127.0.0.1；③写盘端点要求每次启动随机令牌；④`/dub/save-bin` 与 save 同刀清洗；⑤NIGHT-2 攻击脚本全部转正为回归金测试。
7. **R3-1／R3-3 收口**：船长转达无异议即记"已认可"归档。
8. **复核庭排程**：R4 美学平移属参数级（ear＋双哈希纪律即可，免庭）；唱片落仓由 RELEASE 轨做 PROVENANCE 对审（即庭）；M-T3 交付**必过庭**。

## §A Track-SOUND · SOUND-R4 平移与落仓

1. **美学平移**：`sound-params.json record.*` 处置域出厂值向 **C 档**平移，**hiss 电平优先淡**；平移后 `cli ear` 全绿、G7 −20±2 LUFS 不许漂；双哈希变动按治理法记录。
2. **hiss 采样率漂移提级同轮**：±0.4dB@44.1/48k 修复（带内归一噪声源或等效），修后 G3/G7 双采样率复核。
3. **针嗒探针法典化**：复核庭那套"静默唱片＋全床静音重演卡碟"探针写成金测试（防 tick 回归哑火）。
4. **出厂唱片落仓流水线**（候船长终裁名单，先备好管道）：`sound/records/` 装载器＋PROVENANCE.md 模板＋首启征询流程（§0.2 文案与哈希校验）＋Releases 资产打包脚本。船长名单一到，当日落仓。
5. **淘碟指南**：`docs/records-guide.md` 首版（人类 CC0 区／AI 明示区两栏，血统条款原文置顶）。
6. 台账落档：实听排序＋R3 身份终裁＋两项收口记案。
