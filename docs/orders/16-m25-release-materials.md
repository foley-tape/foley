# FOLEY 施工令 · M2.5 发布物料轮（码头段）

> 三轨分工照旧：§A=RELEASE（主责本轮）、§B=STAGE、§C=SOUND（轻）。输入件：《FOLEY 发布工具箱》＋ `audit/night2/COLD_READER.md` ＋ GATE.md。
> 总纪律：**发布时刻每句话为真**；素材诚实条款覆盖一切公开物；本轮一切交付以 GATE.md 条目销账为终点。

---

## §A Track-RELEASE

1. **README 真话补丁**：对照 COLD_READER 逐条销账——每条发现要么修正 README/产品，要么进工具箱 §5 弹药库备防，**零悬置**；与工具箱口径冲突处上报架构师裁（勿自调口径）。测试数/版本号等一切数字改为脚本注入；隐私段落更新"零静默网络＋明示唱片下载"句；新增 Records 段（血统条款一句＋艺术家署名【槽】＋淘碟指南链接）；状态清单更新至实况（live/声音/接带全✓，多轨/托管标 planned）。
2. **公开镜像执行**：按已裁镜像策——公开库 `foley-tape/foley` 以干净历史首推，私库存全史；重媒体不入公开史（Releases 挂载）；推前跑 NIGHT 系隐私回归于**镜像内容**（不是只跑私库）。
3. **`npx foley` 冒烟矩阵**：干净环境（无缓存）三路——同意下载唱片／拒绝走房间层／离线环境；三路皆须秒级起播且零报错。
4. **publish 演练**：dry-run 全绿＋tarball 体积与内容清单人工过目（唱片确不在内）；`private:true` 保持——**拆闸动作只由船长亲手执行**。
5. **demo 页发布**：接 §B 产物，GitHub Pages 绑 foleytape.com；页脚三链（repo／npx 一行／淘碟指南）。
6. 交付：GATE.md 全绿记分牌＋D-1 清单逐项截图归档。

## §B Track-STAGE

1. **demo 页**：只读舞台＋内置一卷示范带（真实 storm 蒸馏带）。**发布前置审计**：该卷先过 redact 级复核＋NIGHT 隐私攻击脚本，通过方可入页。首屏横幅一句："This is a recording — run `npx foley` to hear your own."；声音按电源钮后出（既有仪式）；无任何写盘/导出（demo 是橱窗不是车间）。
2. **hero 实拍**：按工具箱 §6 剪辑指导出主片（MP4＋GIF）与次片；卷号入 meta（素材诚实）；README hero 位替换。
3. **妈妈测试·真人一测**结果随交付归档（若 M-T3 轮未完成）。

## §C Track-SOUND（轻）

1. 出厂唱片 Releases 资产最终打包＋哈希清单（首启征询文案与工具箱口径一致）。
2. 淘碟指南终稿（血统条款置顶、人类区/AI 明示区、艺术家致谢段【槽】）。
