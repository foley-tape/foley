## 0. 提点六条裁决（全批）

1. **交付即合并法（入治理正典）**：轨分支交付＝已自合 main；FEEDBACK 必记合入的 main 哈希；"交付"的定义 = **船长在 main 上看得见**。幽灵期就此关闭。
2. **影子的影子（入流程）**：每条新影子指标上岗前，先在已知条件下验证读数符合构造预期，再采数。审计夜"预注册"精神画侧延续。
3. **恒迟影子直转 active**：delay 恒等式在 frac<1 时数学恒等于 50ms——"对恒等式做统计观察是对数学的不敬"，此句照录入档。针速比与体温两条照旧候两轮。
4. **像素账（替代笼统 CPU%）**：体温预算按层记"每帧触像素数"（镜头／走纸／器件／合成各一行），可设计、可预算、可跨机对比。
5. **多机架工作假设预批**：共享镜头层＋共享房间光（N 台机器一室一镜），预算模型 ≈3.6%＋(N−1)×1.8%；各机各带镜头之路否决。终裁随预告片轮携实测定稿。
6. **昨日的卷：批准接通**。日带轮转命名定死：`runs/live-YYYY-MM-DD/`（本地日界），prune 保留 7 日；舞台加 `?tape=yesterday` 回放入口（半天）。此仓即预告片高光选段的原料仓。

## 2. 仓库卫生法（自声音轨 §三.6 升格全仓，本令载入治理正典）

1. **易失路径禁入仓**：任何指向 /tmp、会话目录等易失位置的符号链接不得提交（本案：node_modules 软链指向他会话 /tmp，typescript 凭空消失）。
2. **共享基建变更申报**：node_modules、tsconfig、launch 配置等共享件的变更，提交信息必须显式声明，另一轨知悉。
3. **轨间配置隔离**：`.claude/launch.json` 类文件各轨各持，禁互相覆盖。
