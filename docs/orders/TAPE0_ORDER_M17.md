## 1. verdict/2 规格

### 1.1 判据生命周期（新增机制，采纳冰箱 #5 并推广）
每条判据带 `status` 字段：
- `informational`：记分不拦路。**新判据首轮一律此态**（判据试用期，入宪）。
- `active`：硬判据，拦路。
- `retired`：退役，附 `reason`，扫参器跳过计分但仍**记录**实测值（留证据链）。

### 1.2 具体修订
- **storm.resolveOnOpportunity → `retired`**，reason："迷路型风暴的解脱不是离散事件；泄能机制正确性已由金测试＋jam 带（真破卡碟×2）覆盖。消散的表达属于 v1 连续层（张力回落/保真恢复/天空放晴），非第四音——现实中雨停无人奏和弦。"
- **battery 级覆盖注记**（写入 verdict/2 头注）：三条泄能通路的真实数据覆盖现状——破卡碟 ✅(jam)；test 转绿 / SAVE 仅金测试覆盖，**留缺口记录**：待出现高张力处 test 转绿或提交收尾的磁带时补一卷。
- **smooth 带 v2**：`dutyTlt30 ≥ 0.985`；**新增 `peakT < 0.50`**（平静之日永不见雨——这才是真正的感知主张）；`rainR ≤ 0.05`；`stuckEdges = 0`。理由入注：原 99% 把判断力量化到了荒谬精度（差 3.5 秒），且那次 0.405 尖峰源于一条约两分钟才失败的命令——引擎算对了。
- **金时刻 v2**：
  - landmark 增加两种能力：`selfLocating`（自定位窗口）与 `precondition`（前置条件不满足 → 记 **N/A，不算 FAIL**）。
  - **L2 重写**为自定位：`kind: decayAfterClear` —— 在最后一次 STUCK_CLEARED 之后，寻找首个时长 ≥120s 的**无充能区间**；断言该区间内 T 非增且相对下降 ≥25%；找不到这样的区间 → N/A。（旧 L2 的固定时间窗是对旧物理画的，已被参数漂移误伤——教训：**能自定位的金时刻不随物理漂移，只随磁带冻结**。）
  - 全部金时刻**再保持 informational 一轮**，v1 进场时转 active。

## 3. 盲听验收（M2-EAR，v0 的人耳闸门）

### 3.1 协议
1. 施工方将五个探针页复制为 `probe-A.html` … `probe-E.html`，映射**随机化**；映射表写入 `runs/ear/MAPPING.sealed.md`（船长答完前不得打开）。
2. 匿名页内不得残留带名/卷号/日期等可推断信息（标题、文件名内嵌字符串一并清理）。
3. 船长逐卷播放（每卷听 2–5 分钟，可拖动），**先答题后开封**。
4. 结果 + 映射对照写入 `runs/ear/EAR_REPORT.md`。

### 3.2 听感表（船长填写，每卷 A–E 各一行 E1–E5，最后答 G 组）
- **E1 盲猜**：这是哪种日子？静默 / 顺风 / 忙碌 / 卡碟 / 风暴
- **E2 回看冲动**：听着它，你想回头看屏幕的频率？从不 / 偶尔 / 频繁（注：这是校准信息不是好坏——silence 期望"从不"，storm 期望"频繁"）
- **E3 跳针**：没听到 / 听到且恰当 / 刺耳
- **E4 和弦**（若有）：无感 / 舒服 / 如释重负
- **E5 最"不对"的一声**：时间点＋一句描述（可空）
- **G1**：五卷里最好听的？最难听的？
- **G2**：愿不愿意让这套声音陪你干活一下午？愿 / 看情况 / 不愿
- **G3**（自由格）：任何想说的。

### 3.3 验收判据（v0 声音通道端到端）
- **E1 盲识 ≥ 4/5**，且
- E5/G3 中无阻断级条目（"这声让我想关掉"级别）。
- 通过 → v0 声音通道验收 ✅；未过 → 逐条进 EAR_REPORT，判据/参数携人耳证词进 verdict/3 裁。
