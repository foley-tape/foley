# TAPE-0 施工方手记 · M1.6 + M1.6-A

> 我（施工方）的**主观**思考，与 `FEEDBACK.md`（客观交接）分开。把执行中撞见的、**超出施工令**的判断摆上台面，替你提前排雷。

## 一、复盘：这轮学到 / 印证的三件事

1. **"实验推翻假设"这次真的发生了，而且是好事。**
   你在 M1.6-A §5 提前给了预授权（零全绿也进 M2），像是早就料到判据可能拦死乐手。结果 0/144，两条判据被实验证伪：storm 的 RESOLVE、smooth 的 99%。**如果没有 §5 这条闸门修正，我会卡在闸门、又出一轮 CSV、你又隔空调参——正是你想避免的死循环。** 这条修正的价值，这轮直接兑现了。

2. **法官（verdict）从硬编码抽成数据，立刻暴露了判据的脆弱。**
   把 bands/landmarks/雨量阈值搬进 `verdict.json` 后，storm 的 4 条判据里有 1 条（RESOLVE）在**任何参数下都过不了**——这在硬编码时代会被淹没在"调参再看看"里。判据一旦可被扫参器批量证伪，它的"假设"身份就藏不住了。**判据即数据，不只是治理，更是让判据接受证伪。**

3. **换尺（雨量 R）是这轮唯一"够到"storm 的东西。**
   占空比对阵发风暴结构性不友好（M1.5 时 storm 占空 1.5%，怎么调都上不去）。换成雨量 R 后，storm 一下能到 R=1.88、占空 17.9%——**尺子换对，storm 的"形状"就对了**。这印证了手记二.2 的判断：占空是错尺子。剩下的 RESOLVE 是另一层（尺子对了，事件本身不存在）。

## 二、我看到、但没擅动的（留给你裁）

1. **迷路 vs 出血：这轮用数据坐实了，且出血型一卷都没有。**
   我扫了全库 63 卷找出血型 storm-b（RUN/WRITE 主导失败）——**零候选**。所有真风暴都是迷路型（fetch/read 疯狂翻找）。这不只是"缺一卷样本"，可能是**这个用户的真实工作里，风暴就长这样**——出血型（跑挂/写坏）很少累积成风暴，翻找型才会。如果迷路型是主流，那 RESOLVE、三音、权重都该按迷路型来设计，而不是按"测试驱动"的出血型直觉。**这是 #1 决策点的根。**

2. **RESOLVE 多态化仍不够，因为它假设"解脱是个事件"。**
   我把 RESOLVE 拓到测试/提交/破卡碟三态，但迷路型风暴的平息**不是任何离散事件**——它就是慢慢不再翻找了（卡碟 expiry 批量消散）。真实的"雨过天青"在这类风暴里是**衰减**，不是**一声和弦**。要不要承认"消散也是一种平息"、给它一个（不同于 RESOLVE 的）微弱下坠音？这会碰你"消散≠解决"的纪律，但迷路型风暴逼着重新审视它。

3. **smooth 当"哨兵"用，代价是 sweep 永远 0 全绿。**
   你选 98.9% 的卷当反向卡尺，意味着**只要网格向上（救 storm），smooth 必红**。于是"全绿冠军"这条路（§4.3）在当前阵列下**永远走不通**，每轮都落到帕累托（§5）。这不一定错——但如果你想让 sweep 偶尔能真·全绿、给正反馈，就得让 smooth 有"喘口气"的余量（阈 98.5%，或选卷时留 0.5% 缓冲）。**否则冠军规则 §4.3 是具文。**

4. **provisional 参数会不会"临时变永久"？**
   READ 0.21/fD 0.4 已落盘。它是帕累托最优，但它**没让任何一条 storm/smooth 判据从红转绿**（只是让 storm 的其它三项达标）。风险：M2 用人耳听了觉得"还行"，于是 provisional 就默认转正，而 RESOLVE/smooth 两条红判据**永远没人回头裁**。建议给 provisional 参数**挂一个到期复核**（M2 人耳结论回来后必须重裁 #1/#2），别让它躺成正式。

## 三、给你的更大决策点（按"越晚越贵"排序）

| # | 问题 | 不定的后果 |
|---|---|---|
| 1 | **迷路型是不是主流风暴？** 若是，整套 RESOLVE/三音/权重该按迷路型重设（出血型全库无样本） | 决定 M2 之后所有校准的世界观；定得晚要重设判据+三音 |
| 2 | **"消散"要不要算一种平息**（给 expiry 批量清空一个下坠，区别于 RESOLVE 和弦）？ | 决定迷路型风暴能否"过境"；牵动"消散≠解决"纪律与第四音禁令 |
| 3 | **验收判据里"结构性不可达项"怎么治理**？ verdict.json 有了，但"这条判据在任何参数下都过不了"需要一个显式状态（infeasible），否则每轮扫参都白撞一次 | 判据可被证伪却无"退役/标注"机制，浪费每轮算力 |
| 4 | **金曲线 vs 金时刻**：这轮金时刻 L2 在新参数下失效了（窗口是对旧物理画的）。金时刻虽比区间锋利，但**也会随物理漂移而失真** | 若下轮把金时刻转硬判据，漂移的 landmark 会误杀好参数 |
| 5 | **M2 人耳判据要不要也数据化**？现在 M2 靠"听感"，无结构。要不要让船长/你听完后填一张结构化观感表（回冰箱/verdict）？ | 人耳结论无载体，M2 的信息增益会随对话流失 |

**我的倾向**：#1 先定（迷路 vs 出血是根）；#2 紧随（它决定 storm 能不能真·绿）；#3 给判据加 `infeasible` 标注，最省未来的冤枉算力。
