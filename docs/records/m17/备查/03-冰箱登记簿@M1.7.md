## 1. episode 软复位（跨段余温）
- **来源**：M1.6-A §1.二.5（缓办入冰箱）。
- **现状**：v0 硬复位维持——SESSION_START 把 S/rep/卡碟/wow/天气/针全部归零。"一坐＝一个叙事单元"是有意的建模选择。
- **草图**：跨段携带 `S' = min(0.15, 0.2 × S_end)`，让 resume 一场苦战时留一点情绪残温。
- **解冻条件**：确认"多日续跑的情绪连续性"值得建模（v1 舞台期）。

## 2. StatePacket `texture` 字段（风暴质地）
- **来源**：M1.6-A §1.二.3（入冰箱，v1 裁）。
- **现状**：协议 schema 冻结、本轮零改动。texture 不入 StatePacket。
- **草图**：texture ∈ {wander(迷路)/bleed(出血)}，由失败动词分布派生。**注**：本轮扫库证实全库风暴皆迷路型、出血型零样本——texture 现在只有一个取值，建模价值待出血样本出现。
- **解冻条件**：出现出血型风暴样本 + 协议解冻签核。

## 3. 质地分音（迷路/出血不同声音）
- **来源**：M1.6-A §1.二.3 + 手记二.3。
- **现状**：不设——v0 三音（拨弦/和弦/跳针）不分质地；且"不设第四音"禁令在。
- **草图**：迷路型跳针 vs 出血型跳针用不同音色/包络区分（仍算"跳针"一音的两种质地，非第四音）。
- **解冻条件**：#2 texture 落地后，v1 声音层。

## 4. 全脱敏分享带（默认可分享形态）
- **来源**：M1.6-A §1.二.4。
- **现状**：**机制已建**——`distill --redact` 把 errClass 换聚类哈希（e+8hex），产零明文分享带（金测试 ⑬）。**默认仍不脱敏**（本地抽检需人读错误模板）。
- **草图**：把"可分享"做成一条完整管线（v2 托管分享期）：redact + 元数据剥离 + 一键导出。**注**：v0 禁"导出/分享功能"，故此项止步于 `--redact` 本地开关，导出/分享入冰箱。
- **解冻条件**：v2 托管分享里程碑 + 分享功能解禁。
