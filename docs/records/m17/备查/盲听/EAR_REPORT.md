## 二、逐条证词（船长原话要点）

1. **silence/smooth/busy 差距太小**：三者拨弦频率差不多，只有三卷对比才判得出。→ 声音**没编码"强度/紧张"**，只靠事件密度这一弱信号。
2. **storm/jam 听到"好几种声音"**（尤其风暴），无乐理背景，不确定是多音叠加还是频率拉伸太大误听。→ 实为三种质地并存：**OK 拨弦(高音)/FAIL 拨弦(低音)/跳针(噪声脉冲)**；风暴 FAIL 多 → 低音多 + 跳针×9。非 bug，是"成功=高、失败=低"的音高编码在起作用。
3. **提示声像手机"叮"**：中规中矩，不难听但也不讲究。→ 音色是通用通知音，未经雕琢（**无阻断级**）。
4. **突变 → 想看**：声音突然变化时想回看屏幕，尤其**频率变大/急剧变化**时更想看。→ **正中设计意图**（声音替眼睛值守；升频那口"警觉"尤其有效）。
5. **[关键] 常态声应融入 lofi 背景**：新手初听会被每一处异同声吸引去看（过度告警）；重要的是让**正常运行的声音沉入背景 lofi，只让偏差冒头**。
6. **[关键] 看不懂曲线↔声音关系**：有时曲线很陡却无声，有时线几乎平却频繁出声。
