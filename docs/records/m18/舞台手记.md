# 舞台手记 · M-S1

> 与施工方手记同格：品味发现、宪法修正案草稿、器件与字段绑定中的现实修正。

## 现实修正（按宪法执行＋起草，未先斩）

**一、curve.csv 是 10Hz，施工令说 20Hz。**
`cli/replay.ts` 的 `SNAP_MS=100`；引擎注释设想的是 20Hz 广播。舞台侧照令重建到 20Hz 网格（线性重建），但这是重建不是采样。**提请 Track-FIX**（不伸手）：live 落地时广播真 20Hz，或把 SNAP_MS 降到 50——针的弹簧-阻尼是引擎的心跳，采样密一倍，针就少一半"数据台阶"。

**二、curve.csv 无 pendingAsk 列。**
绑定表说琥珀管吃 `pendingAsk` 字段，可 curve.csv 八列里没有它。读 `engine/index.ts` 的 `phaseOf`：`done` 先判，之后 `pendingAsk ⇒ WAITING`——**WAITING ⇔ pendingAsk 是同源等价**，故以 phase 推导，不算二次发明。但有一条缝：DONE 期间若 pendingAsk 尚存，画侧读不到。**提请 Track-FIX**：curve.csv 增 `pendingAsk` 列（产物格式，不动冻结协议）。

**三、蒸馏带有拼接，回放钟得学会咽口水。**
storm 的时间轴横跨十一天，中间二十个小时级空洞。策略：单步 dt 夹到 400ms 上限，>2s 记一道**接带痕**，在走纸上画出来（斜细痕＋浅高光）。空洞不假装播放，也不假装不存在——磁带宇宙对拼接的诚实。屏录里 1092s 附近那道痕就是它。

## 宪法修正案草稿（供架构师裁）

**草案 A（材质法附注）**：三色标（基/高光/阴影）宜明示为"材质的固有色，受光法调制"。M-S1 实测：宪法三色平铺在大面积上，暗场立不起来；把渐变尾端压到 #67583E（较 #7E6E4F 再深一档）后，面板才肯从黑暗里浮出来。色标未弃，只是让它们站进了光里。

**草案 B（器件法附注·双宝石条款）**：绿宝石承载 DONE 常亮与 RESOLVE 一闪两义。自辩：二者同族（完成/解决），且一为持恒一为脉冲，不同频段不打架——恰如带轴的"转速+稳定度"。若架构师认两义超编，RESOLVE 之闪可归还给未来的计数轮跳位。

## 品味发现

- **纸空间记录是免费的正确性**：墨点记的是纸位不是时刻，于是 8× 铺纸与 1× 实走在纸上**全等**——屏录的预热段因此不留指纹。纸带即时间轴这条教义，落到数据结构上就是一行字段选择。
- **禁缓动反而省了代码**：针的全部生命力来自引擎的弹簧-阻尼数据。渲染端删掉一切 transition 之后，storm 段针的微颤（wow 的余波）自己浮现了——保真度即张力，在画侧同样成立。
- **钨丝分频**：灯的热惯性走真实时间、走纸走舞台时间。倍速回放时纸跑得飞快而琥珀呼吸如常——机器在快进，可它对人说话的节奏不变。这条没写进宪法，但它应该是对的。
- **静态噪点先上岗**：镜头法（颗粒 shader/漂移）归 M-S2，但"暗场渐变必须抖动防色带"等不得——4K 下无噪点的暗角是一圈等高线。已以 3.5% 静态噪点垫底，M-S2 换动态颗粒时替换。
- **型号丝印**：TAPE-0 写作 `TAPE · ZERO`。数字永不见于面板明处——包括机器自己的名字。
