## ① A1 · 邮箱 PII 漏进默认蒸馏带（NIGHT-2 P1）

**病灶**：`normErr`（parse.ts）有凭据/路径/hex/token/数字规则，唯独无邮箱规则。邮箱恰好全漏：无 `=`/`:` 不触发内联凭据规则，`@` 又把长 token 切碎躲过 16 位规则。`errClass` 是默认带唯一由**输出**派生的字段，故 README「outputs never stored」在此名不副实（审计 A1 原话）。redact 分享带会哈希掉，所以只默认带带毒。

**修法**：邮箱规则**置于规则链首位**——若放后面，先行规则会把邮箱切碎（如 `-p` 规则吃掉一段），残片漏抹。规则：

```ts
.replace(/[a-z0-9._%+-]+@[a-z0-9.-]+\.[a-z]{2,}/g, 'EMAIL')  // 居首：防后续规则切碎漏抹
```

（输入已先 `toLowerCase()`，故规则只需小写域。）

**冒烟实证**：
```
输入: Error: mail to Alice.Smith+dev@Example-Corp.co.uk failed at /Users/x/y
输出: error: mail to EMAIL failed at PATH
```

**常设闸门**：`golden/privacy.redteam.test.ts` 新增测试「隐私·默认带 normErr：邮箱 PII 抹为 EMAIL（NIGHT-2 A1）」——三形态向量（普通 `alice.smith@example.com`／大写＋加号标签 `Bob_Jones+dev@Corp-Mail.example.ORG`／多级域 `x@sub.foo-bar.co.uk`），三断言（全量明文不存活／局部 `local@` 不存活／`EMAIL` 占位在场）。与七类对抗密钥同册永久在跑，补上审计点名的「金测试无邮箱向量」缺口。
