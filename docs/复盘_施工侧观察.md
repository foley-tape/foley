# 复盘 · 施工侧观察与建议
记录：施工终端 · 2026-07-10
定位：应船长之命的复盘——**架构师未提、但我在执行席上认为值得改**的点。全部来自实战伤疤，不是清单体操。供架构师择取入宪/入账；在被采纳前，本文不构成任何施工授权。

## 〇、一段话复盘

这一程走了三个纪元：**手令时代**（十四道手令逐令施工，质量靠架构师看图命中）→ **场景板重构**（013 停工重建，把"贴入/替换"的增量式连根拔掉——整机一次布光渲染成板，只有会动的东西是活层）→ **账本时代**（LEDGER 接管，一次一项、船长机证据才算完工）。贯穿始终的一条主线是：**"验收条件≠产品条件"这一种病，前后发了七次**——自测全绿实机全崩、headless 截图把 canvas 冻在首帧、遮挡节流把 CPU 演成 0%、idle 百分比追错口径……每一次都是"我以为在测产品，其实在测测量工具"。下面的建议大半是对这条主线的制度化回应。

## 甲、工作范式（六条）

### 1. 证据分级表：把"什么工具能证什么"钉成一页表
**伤疤**：headless 截图对 2D canvas 恒为首帧（墨线"消失"追了一小时）；`--dump-dom` 下 rAF 根本不走；窗口被遮挡时 Chrome 停止合成，CPU 采样得出 0% 的假清白；自测绿→实机崩历史上七次。
**建议**：`docs/` 里立一页矩阵——主张类型（静态视觉／运动视觉／声音／性能／交互）× 最低证据等级（headless 截图 → 自泵诊断口 → 真浏览器 CDP → 船长眼耳）。任何"已验证"必须标注用的是哪一级。账本的"船长机证据"条款是终点，这张表是通往终点的梯子。

### 2. 修前必留病理标本（基线先行律）
**伤疤**：P0-1 若没有修前的 53%/37% 基线和修前风暴录屏对照，"修好了"就只是一句话。这次差点跳过基线直接动刀。
**建议**：成文一条：凡修性能/质量/行为，第一动作是把病症录下来（截图/录屏/数字），路径入账。成本两分钟，收益是"修复可证明"。

### 3. 代理指标条款
**伤疤**：追"待机 CPU 个位数"烧了半天，最后发现单核 38% = 整机 3%、渲染管线 trace 近空转、帧医生全绿——**口径没定义的百分比是陷阱**。真验收（240s 录屏零冻盘 + 长帧计数）半小时就跑完了。
**建议**：验收判据一律用**用户可感规格**（录屏、帧医生计数、耳朵、眼睛）；百分比/毫秒只做诊断线索，不做验收关卡。判据入账时必须写"怎么量、在哪量、口径是什么"。

### 4. 发现搁置箱（一次一项纪律的配套设施）
**伤疤（本次最痛）**：P0-1 期间我在录屏里**看见过**盘在长卡拍段的怪样，但账本没有"发现"停车位，一次一项纪律又禁止顺手修——结果既没修也没登记，最后是船长替我抓的。"顺手的好意是回归的温床"是对的，但**没有停车位的纪律会把发现挤丢**。
**建议**：账本加一节"发现栏"：一行一发现（现象+哪里看到的），不修、不展开、不算领项。每轮领 P0 前先扫一眼发现栏。

### 5. 诊断口注册表
**现状**：`?autoplay` `?audiotap` `?perf` `?shadow` `?hud` 已经五个诊断口，散在四个文件的注释里。它们是验收的生命线（真浏览器采集管线全靠前两个），但没有任何一处总账。
**建议**：`docs/诊断口.md` 一页登记：参数、打开什么、给谁用、"保证不进正常路径"的实现位置。新诊断口必须登记才算存在。

### 6. 账本应住进仓库
**伤疤**：LEDGER 首次落在 `~/Downloads/` 被 macOS TCC 咬死（无 macl xattr 的文件终端整体不可读），一来一回耗一轮对话。真相现在住三处：账本（Downloads）、我的记忆库（本机私有）、仓库文档——三处会漂移。
**建议**：`FOLEY_LEDGER.md` 入 repo 根（版本化、终端可读写、TCC 免疫、git 历史即审计），Downloads 只放签发时的副本。我的记忆库只存"指针+个人勘误"，不存账。

## 乙、布局与代码架构（六条）

### 7. stage.css 的双纪元尸体
**现状**：场景板重构后，stage.css 里约三百行旧机器规则（#panel/#deck/bezel/.len-tab 钢键皮/np 胶囊…）成了尸体，靠 plate.css 后加载压制。已经咬过一次（DUB 键幽灵 box-shadow、np 胶囊 translateX 位移都得逐条反制）。
**建议**：签章合入后做一次"清葬"：把活着的部分（rack/play-breathe 关键帧/wire-tag/hud）搬清，死规则删除。收益是拆掉"幽灵覆盖"这层常驻心智负担。

### 8. 坐标单源
**现状**：板坐标真相在 `plate.coords.json`，但 plate.css 的百分比和 machine.js 的内联样式是**手拷**的两份副本。板若重渲且相机动过，三处要人肉同步——必漂。
**建议**：小构建步（或启动时 fetch）把 coords 生成 CSS 自定义属性（`--pos-vu: …`），布局只引用变量。一源三用，重渲零同步。

### 9. 演示带的策展元数据
**伤疤**：demo 取景点 `SEEK_S=920` 是拍脑袋定的，结果开场 24 秒后一头扎进 **192 秒连续卡拍**——把整盘带最难看的三分钟当了橱窗首秀（本次船长报案的放大器）。
**建议**：每盘 fixture 配一行策展元数据（可看窗口、情节点：起转/风暴/跳针/收尾），demo 取景从元数据读。橱窗放哪一幕是**内容决策**，不该埋在常量里。

### 10. 胶片条的"清晰度—内存—帧率"三角（已知取舍登记）
**现状**：条帧 720→512 后解码纹理 500MB→240MB（风暴冻死首嫌已除），代价是 retina 慢速下盘面略软。若船长将来嫌糊，正路是 LEDGER P0-1① 已点名的 WebGL 纹理图集（单纹理驻留、逐帧 UV 抠格、可上 mipmap），不是回大图。登记在此，免得将来往回走。

### 11. 声资产"上桥可诊"（P0-2 的页面侧半边）
**观察**：P0-2 的诊断方向若为真（风扇噪=唱片未上桥、只剩房间层），根因是**资产加载失败静默降级**——机器装作一切正常。`foley doctor` 是终端侧答案；页面侧机器也该物理表达"唱片未上桥"：走带牌曲名位留白/一盏暗，而不是让用户用耳朵猜。属 E5 状态可诊的声音分支，施工时机随 P0-2。

### 12. serve.mjs 的资产口径（挂账不急）
**现状**：assets 已 8MB+（两条胶片 5MB），serve.mjs 无 cache/etag。本地无感；出屋（demo 分享/npx 包）时会疼——首开全量下载且无缓存复用。挂账：出屋前给资产加内容哈希名+长缓存。

## 丙、编舞层的一条新律

### 13. 长值班测试（本次盘抽搐案的教训条）
**案情**：卡拍编舞（前冲—弹回，0.92s 一拍）是按"跳针几秒钟"设计的；storm 真带给了它 **192 秒**——按秒设计的动作值了三分钟的班，节拍器式重复把"会话卡住了"演成了"播放器坏了"。
**建议成律**：任何"进入某状态就改变行为"的编舞（卡拍/呼吸/信号灯/示能），设计评审必答一问：**"此状态最长可能持续多久？届时这个动作还成立吗？"** 编舞要么自带节律变化（挣扎—歇），要么自带值班上限（衰减/释放条件）。已顺此律修：卡拍 v2 = 4.6s 宏周期两记挣扎+静伏，且断电即松（防 CLEARED 缺席闩死）。

## 丁、工程勘误录（个人记忆 → 仓库共享）

以下勘误现在只住在我的（本机）记忆库里——记忆是单机的，仓库才是共享的。先收录于此，建议将来独立成 `docs/工程勘误录.md`：

| 勘误 | 一句话 |
|---|---|
| headless 双坑 | `--virtual-time-budget` 下 rAF 不走（须诊断口自泵）；2D canvas 不再向合成器上屏（截图恒首帧，墨账用 `getImageData` 验真） |
| node 内置 WebSocket | undici 实现：不持事件循环（无定时器的 await 空转退出）；单帧 >1MB 或累计 ~7MB 即 1006——CDP 采集一律用 npm `ws` + 大回包分片 |
| 遮挡停合成 | macOS 下窗口被遮挡 → Chrome 停合成 → screencast 饿死、CPU 假 0%——采集加 `--disable-backgrounding-occluded-windows` + `Page.bringToFront`，并先验 `visibilityState` 与帧计数在走 |
| h264 偶数律 | `--app` 窗高被标题栏啃成奇数 → libx264 拒收，`crop=floor(iw/2)*2:floor(ih/2)*2` 兜底 |
| 金属只吃 glossy | Blender 里 metallic=1 材质对 `visible_glossy=False` 的灯完全免疫——牌灯 34W/300W 全黑皆此因 |
| TCC/macl | ~/Downloads 里无 `com.apple.macl` xattr 的文件对终端进程整体 EPERM（读/改名皆拒）——工作文件入 repo |
| 相机锁死=坐标即常量 | 场景板相机锁死后，`world_to_camera_view` 导出的归一化坐标可当常量用于 CSS 对位——"动态层靠算不靠瞄"的根基 |
| 混合层×动画 | mix-blend 元素一动（或其下层一动），整幅重混合每帧付账——动画元素要么撤 blend + `will-change` 上合成器，要么与被混合内容进同一缓存子树 |
