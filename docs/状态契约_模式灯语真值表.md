## 一、病理（为什么需要契约）

今日「机器在不在录」有**四个各说各话的真值源**：
1. serve transport 广播（`t.live ∧ t.phase==='PLAYING'` → `body.rec-live`）——serve 死则无人撤，**尸体上照亮**；
2. 页内 `liveActive`（main.js 闭包，mount/停机翻转）——观察者语义，与 rec-live 可分裂；
3. live 连接态机（live.js：connecting/live/lost/gone）——只管**链路**，不知 producer 死活；
4. selector 本地档位（off/test/on）——素面 OFF 与 REC 红已实测同帧（右耳 D-5 铁证）。
另：producer（真 Claude 进程）死 26 秒 REC 照亮（左耳 claude-kill 铁证）——机器监听的是尾随器，不是生产者。
**D2 增补第五病**：即使 derive 归一，**引擎包旁路**（Lamps 每收一包用原始 `pkt.pendingAsk` 覆写琥珀）令 derive 的 R2 门在运行时被绕过——纯函数金测全绿而真机说谎。单写者非到渲染点不算数。

## 二、导出态（单一事实源）

页面一切**状态性渲染**（灯/指针/SIGNAL cue/盘走停/纸走停/绿宝石）自本契约导出态读取，禁止旁路直读五源、禁引擎包旁路改写。导出态由 `deriveMachineState(inputs)` 纯函数计算（居 stage/js·两页同法）：

```
inputs = {
  power:    'off' | 'test' | 'on',          // selector 本地档位（POST 演出期由 post 编排借用）
  phase:    'EMPTY'|'CUEING'|'PLAYING'|'PAUSED',  // transport 广播（serve 权威）
  sourceKind: 'live' | 'session' | 'factory' | 'none',   // 上机带血统（transport.loaded 派生）
  link:     'connecting' | 'live' | 'lost' | 'gone',     // live 连接态机（仅 sourceKind=live 时有意义）
  producer: null | 'alive' | 'dead' | 'ended',  // 工单 2 心跳四值（v1.1）：null=unknown（无 PID·永不判死）/
                                            // alive（kill -0 在场）/ dead（PID 心跳消失=猝死）/ ended（session-end=善终）
  pendingAsk: boolean,                      // transport 保活字段（rule 4·唯一读点）
  done:     boolean,                        // v1.3 新：引擎 DONE 态（页侧包相位 pkt.phase==='DONE'）——settled 唯一输入源
  post:     null | PostScript,              // POST 演出期借用（借走必还）
}
```

```

```

## 三、导出规则（法条形式）

R1 **红灯八闸**：`recording = (power=='on') ∧ (sourceKind=='live') ∧ (phase=='PLAYING') ∧ (link=='live') ∧ (producer∉{'dead','ended'})`。
  ——五因子缺一即灭。推论（铁例执法）：power=off/test ⇒ REC 永灭；link=lost/gone ⇒ REC 灭；producer=dead/ended ⇒ REC 灭（猝死与善终都不再录，但只有猝死打死相）。
R2 **琥珀单义（v1.3 收紧）**：`asking = recording ∧ pendingAsk`。
  ——**琥珀不先于红**：ASK 必发生在录制中（live·PLAYING·链活·producer 未死未善终）。live 恒 PLAYING（活流不可暂停·PAUSED 是回放概念），故此式对真机 live-ASK 恰好成立。呼吸**整个等待期**（pendingAsk 为真的每一帧）；ASK_CLEARED 即熄，无淡出戏。**清零三类旧非法琥珀**：非 live 源、`ended`、未录（phase≠PLAYING/断链/断电）一律不许琥珀（等一个死人/画外人/未开录的问题都是谎言）。
R3 **LINE 基底**（v1.2）：`linkLit = (power!='off') ∧ (sourceKind=='live' ⇒ link ∈ {connecting,live,gone}) ? 0.12 : 0`。修宪甲原文照护：≤0.15 红宝石属基底照度不构成信号红。
R4 **死相三词**：`signalCue`：link=lost → 'lost'（Signal Lost）；link=gone → 'gone'（Source Gone·尾随器亡）；producer=dead ∧ link=live → 'dead'（Source Gone 语义族·工单 2）。**producer=ended 无死相**（善终不是死——收工语义归引擎 DONE/歇场）。非 live 带恒 null；OFF/TEST 恒 null。
R5 **绿宝石（v1.3 实装）**：`settled = (power!='off') ∧ done`。常亮=场成了，一闪=一件成了（RESOLVE 瞬态泛光仍归 Lamps 事件层·非机器持续态）。**不受 link/producer 覆盖**——完成是历史事实，不因断链/producer 死撤销；但 power=off 熄（R8 power 压一切）。`done` 输入源＝页侧引擎包相位 `pkt.phase==='DONE'`（收敛点在 main.js 收集器·非 Lamps 私算）。
R6 **走带**（v1.3.1 修·退役 transportRun 导出）：盘走/停归 `power` 与 `phase`——link/producer 异常**不停盘**（带走的是"已收到的过去"，观察者诚实）。**单写者＝实现侧**：回放转台 `replayer.play/pause` 随 transport phase（main.js applyTransport）＋断电 `stopMachine`；此即走带唯一驱动。derive 不再产 `transportRun` 导出——它从无运行时消费者，留着即死输出（席一 D2 复审 #4）。
R7 **POST 借用**：post!=null 期间灯语由 POST 编排接管（借走必还），归还帧必须等于本契约当刻导出态（还灯即对账）。
R8 **优先级**：power 压一切；其次 producer=dead 与 link 异常（R1/R2/R4 已内含）；余下按 phase。

## 五、pendingAsk 供给链（现状勘定＋目标）

现状：`AskUserQuestion` tool_use 落盘 → incremental askOpen（合成 verb=ASK·resolveT=null）→ 引擎 `st.pendingAsk=true`（只有 ASK_CLEARED/SESSION_START 清）→ live 20Hz 状态流每包携带 → serve `transportSetPendingAsk` 跳变广播（rule 4）→ main.js 收集器 `S.pendingAsk` → deriveMachineState → refreshMachineState 喂 lamps。**页面永远只读导出态**：Lamps 不再直读 `pkt.pendingAsk`（v1.3 拆旁路）。
目标补全：权限等待（permission prompt）若不落盘则蒸馏不可见——由钩子域补真相源，其输出仍必须汇入 `transport.pendingAsk` 单点。
