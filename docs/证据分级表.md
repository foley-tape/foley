# 证据分级表
律（LEDGER·增补包 v2 §1）：**一切"已验证"必须标注证据级**。低于最低级的主张只许写"待验"，不许写"已验证"。
背景：七次"自测全绿、实机全崩"的病理全是"以为在测产品，其实在测测量工具"。

## 证据级
| 级 | 器具 | 能证什么 | 不能证什么（伤疤） |
|---|---|---|---|
| **L0** | headless 截图 / `--dump-dom`（virtual-time） | 静态布局、CSS 驱动层当前态（雪碧图 background-position、辉光 opacity、transform） | **rAF 驱动的一切**（虚拟时 rAF 不走）；**2D canvas 后续帧**（不再向合成器上屏，截图恒首帧——墨线"消失"案）；声音 |
| **L1** | 诊断口自泵＋程序读数（?autoplay 喂包、getImageData 墨账、__shadow、doctor 输出） | 逻辑正确性、canvas 本体内容、数据管线端到端 | 合成器/GPU 真实呈现；帧率手感；混音听感 |
| **L2** | 真浏览器 CDP（screencast 录带、captureScreenshot、audiotap 声轨、DOM evaluate、帧医生读数） | 用户将看到/听到的像素与波形；性能的用户可感规格（零冻盘、LONG 计数、dB 值） | 品味（"廉价感/塑料感/难受"）；多日真实使用流 |
| **L3** | 船长眼耳手（真机十分钟、录屏报案、一句话判词） | 品味、手感、完整用户故事 | ——终点级 |

## 主张类型 × 最低证据级
| 主张 | 最低级 | 备注 |
|---|---|---|
| 布局/静帧视觉（位置、颜色、字） | L0 | 涉 canvas 内容则 L1（getImageData）起步 |
| 运动视觉（盘转、墨爬、灯呼吸） | L2 | L0 截图对 rAF/canvas 恒假 |
| 声音（存在、响度、频谱、A/B） | L2 | audiotap 波形＋数字审；"难受与否"归 L3 |
| 性能（掉帧、冻结、开页时延） | L2 | 用户可感规格：录带扫描/帧医生计数/秒表；百分比与毫秒只做线索（代理指标条款） |
| 交互流（点带、退带、手势、迟到者开页） | L2 | 必须走**完整用户故事**——"验收≠产品"第七发（P0-3）：只验组件动≠验开页有史 |
| 数据/纯函数（判定器、解析、参数执法） | L1（金测试） | node --test 即 L1；类型面加 tsc |
| 品味/示能/掉价感 | L3 | 唯一裁判 |

## 使用法
- 证据栏行尾标注：`……（L2：真 Chrome 240s 录带·零冻盘扫描）`。
- 升级不越级：L1 过了才值得烧 L2；L2 过了才请 L3（船长时间是最贵的采样器）。
- 测量器具自身要先验活（visibilityState/帧计数在走、profile 无旧码缓存）——见 docs/工程勘误录.md。
